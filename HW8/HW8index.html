Equal Interval Map



 This map's classification method takes the value range and divides it by the number of classes your provide for the choropleth, presenting the data in equal groups. In my map, the data ranged from 26 to 685. To get the amount of data points in each class, you must do the math as follows (using 7 classes as indicated on the legend): 685-26 = 659 --> 659/7 = ~94. As you can see, there are 94 data points in each set of classes on my map.

Pros:

Best for continuous data sets such as precipitation. The map is easy to interpret, especially when aimed at a nontechnical audience.

Cons:

There will almost always be an uneven spread and distribution of data between classes, hence why there is only one county with the darkest orange, and 46 with the lightest orange.

Natural Breaks Map



 This map's classification method groups data by identifying distinct "natural" cut offs and break points in the data.

Pros:

The data arranged so that there is less variation within each classification, and outliers are grouped together. This methos is best used when presenting unevenly distributed data.

Cons:

This meyhod can create classifications containing widely varying data ranges. It is also difficult to compare two different maps that use the natural breaks method becuase classification ranges are very specific to each data set.

Equal Interval Map



 This map shows the highest average density of people of Hispanic origin per county, as you can see by the large area of dark orange covering the map. Also known as the quantile classification, this method seperates the data into categories based on the number of individual features that are being accounted for. To further clarify, in my example, the "features" would be the counties in Florida (67 of them), and since I am using 7 different classifications, each classification (the different colors on the legend) is distributed to either 9 or 10 counties.

Pros:

This method is best used for data that is evenly distributed. It also helps with emphasizing relative positioning in regerd to the whole data set.

Cons:

Since this method revolves around having a certain number of data points or features in each classification, the data points are not evenly distributed and are likely to have a wide range of values within each class, giving a false impression of actual statistics. Data with very small differences can also be placed in different classifications, which gives the facade that the dataset is more varried than in actuality.
